# Open Day Style Adaptation: CSIE Material Checklist

Reference video type: university department open-day / department tour video.

Adaptation target: 國立臺南大學資訊工程學系 60 秒形象宣傳影片。

Important note: This project should reference the structure and rhythm only. Do not copy the original video's exact narration, subtitles, music, footage, or editing assets.

## 60-second Adapted Structure

| Time | Section | Purpose | Needed Materials |
|---|---|---|---|
| 0-5s | Opening hook | Quickly connect technology and future learning | Campus image/video, laptop opening shot, code screen, title text |
| 5-12s | Department identity | Introduce 國立臺南大學資訊工程學系 | Official CSIE header image or self-made title card, campus background |
| 12-22s | What students learn | Show CSIE learning domains | Coding footage, AI/data/system/network/multimedia keywords, screen recording |
| 22-34s | Hands-on practice | Show that learning becomes working systems | Demo screen, website/app/terminal output, project screenshots |
| 34-45s | Teamwork and projects | Show collaboration and problem solving | Group discussion photo/video, whiteboard, laptop team shot |
| 45-55s | Campus and learning atmosphere | Make the video feel like a real department introduction | Campus photos, library/building images, study environment |
| 55-60s | Closing | End with department name and slogan | Clean title card, official reference/source note |

## Minimum Materials Needed

You can finish the video with these minimum assets:

1. One department title visual:
   - Preferred: self-made title card with text "國立臺南大學資訊工程學系".
   - Optional: `materials/candidate_images/official_csie_header_bg.png`, but official website image permission should be confirmed.

2. Two campus visuals:
   - `materials/candidate_images/nutn_main_building_wikimedia.jpg`
   - `materials/candidate_images/nutn_library_wikimedia.jpg`

3. Three CSIE learning visuals:
   - A laptop/code photo or screen recording.
   - A simple AI/data/system/network/multimedia keyword card.
   - A demo screenshot or screen recording of any program/webpage/app.

4. One teamwork visual:
   - Self-shot discussion scene, hands-only laptop shot, or licensed stock image from Pexels.

5. One closing card:
   - Department name.
   - Short slogan.
   - Source note: "素材來源詳見 GitHub repo / outputs/asset_sources.md".

6. Audio and text:
   - `outputs/subtitles.srt`
   - Optional CapCut voiceover
   - CapCut free/usable background music, or no music

## Visual Style Notes

- Use clean subtitles and short text overlays.
- Prefer 2-4 second clips instead of long static shots.
- If using photos, add slow zoom or pan in CapCut.
- Use the official CSIE header only briefly if used at all.
- Do not use the original YouTube video's footage, music, or exact narration.
