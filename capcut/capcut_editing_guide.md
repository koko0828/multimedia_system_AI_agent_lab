# CapCut 剪輯指南

## 基本設定

- 專案比例：16:9
- 解析度：1080p 或 720p
- 影片長度：目標 60 秒，可接受 55 至 65 秒
- 匯出格式：MP4
- 建議字體：CapCut 內建清楚可讀字體
- 字幕位置：畫面下方安全區，不遮住人物臉部或重點畫面
- 音樂：使用 CapCut 可商用/可使用音樂，或改用無音樂加旁白

## 參考影片風格

可參考 YouTube 影片的「大學 OPEN DAY 系所導覽」形式，但只參考節奏與段落安排，不複製原影片的旁白、字幕、音樂或畫面。改寫後的資工系素材清單在 `capcut/open_day_style_material_checklist.md`。

## 你需要拍的素材

建議每段拍 5 至 10 秒，最後挑最穩的片段使用。

1. 校園或系館外觀：招牌、走廊、教室外觀。
2. 筆電或電腦畫面：打程式、執行程式、看資料圖表。
3. 手部特寫：鍵盤、滑鼠、筆記、白板。
4. 小組討論：同學討論、白板畫流程、看螢幕。
5. Demo 畫面：作品展示、網頁、程式輸出、簡單互動畫面。
6. 校園氛圍：走路、教室、學習空間。
7. 片尾畫面：乾淨背景加系名標題。

## 如果使用網路照片

可以用網路照片完成，但請優先使用 `outputs/asset_sources.md` 中列出的 Wikimedia Commons 或 Pexels 來源，並保留來源與授權。官方網站圖片若沒有明確授權，建議只作為資訊參考，不要直接抓圖當主素材。

在 CapCut 中使用靜態照片時，建議加上輕微放大、平移或淡入淡出，讓它看起來像短影片而不是純投影片。

## 60 秒剪輯時間軸

| 時間 | 畫面 | 字幕/旁白 | CapCut 操作 |
|---|---|---|---|
| 0:00-0:05 | 校園或筆電開機、快速切入程式畫面 | 你想把想法，變成真正能運作的作品嗎？ | 快速推近或淡入，加入主標題 |
| 0:05-0:10 | 系名標題、校園外觀 | 這裡是國立臺南大學資訊工程學系。 | 放大字幕，畫面保持穩定 |
| 0:10-0:15 | 打程式、筆記、課堂/自習畫面 | 從程式設計開始，建立資訊工程的基礎。 | 字幕下方，畫面可加輕微轉場 |
| 0:15-0:22 | 程式碼、資料圖、AI 或互動媒體畫面 | 我們學習 AI、資料、系統、網路與多媒體。 | 用 2-3 個短鏡頭加快節奏 |
| 0:22-0:30 | 螢幕錄影、作品 demo、操作畫面 | 把概念寫成程式，把程式做成可以展示的系統。 | 畫面可加入簡短關鍵字貼紙 |
| 0:30-0:38 | 小組討論、白板、同學看螢幕 | 在專題與團隊合作中，練習溝通、除錯與實作。 | 音樂稍微推進，字幕保持清楚 |
| 0:38-0:45 | 成品展示、手機/網頁 demo、互動情境 | 讓技術不只停在課本，而能連結真實問題。 | 用作品畫面當主視覺 |
| 0:45-0:52 | 校園生活、教室、走廊、學習氛圍 | 在臺南的校園裡，和夥伴一起累積能力。 | 慢一點的轉場，情緒收束 |
| 0:52-0:57 | 系名、學生背影或乾淨背景 | 從一行程式，走向下一個可能。 | 加上 slogan |
| 0:57-1:00 | 片尾標題與素材來源提示 | 國立臺南大學資訊工程學系。 | 顯示片尾與 repo/source 提示 |

## CapCut 操作步驟

1. 新增專案，選擇 16:9。
2. 匯入拍攝素材，先依時間軸排序。
3. 每段素材剪到表格中的秒數。
4. 加入字幕，可以匯入 `outputs/subtitles.srt` 或手動貼上。
5. 若要旁白，可在 CapCut 錄音，照 `outputs/subtitles.srt` 的文字念。
6. 加入低音量背景音樂，避免蓋過旁白。
7. 片尾加上「素材來源詳見 GitHub repo / outputs/asset_sources.md」。
8. 匯出 MP4，確認長度介於 55 至 65 秒。

## 檢查清單

- 影片長度 55 至 65 秒。
- 有字幕或旁白，建議兩者都有。
- 字幕沒有被畫面邊界或 UI 擋住。
- 沒有未同意的人臉特寫。
- 沒有未授權音樂、Logo 或網路圖片。
- 片尾或 README 有素材來源說明。
- 匯出後檔名為 `final_video.mp4`。
