# GitHub and Ecourse Submission Guide

## 1. Create GitHub Repository

1. Go to https://github.com/
2. Sign in.
3. Click `New repository`.
4. Repository name suggestion:

```text
final-agentic-short-video-project
```

5. Set visibility to `Public`.
6. Do not upload API keys, passwords, or private screenshots.

## 2. Upload Project Files

Upload the full project folder contents, including:

- `README.md`
- `SUBMISSION_CHECKLIST.md`
- `GITHUB_ECOURSE_GUIDE.md`
- `report/`
- `agents/`
- `handoff/`
- `traces/`
- `baseline/`
- `capcut/`
- `outputs/`
- `圖片/`
- `materials/`

Most important files:

- `outputs/final_video.mp4`
- `report/Final_Report_S11259035.pdf`
- `README.md`
- `agents/*.md`
- `handoff/*.json`
- `traces/execution_trace.md`
- `baseline/single_agent_baseline.md`
- `outputs/asset_sources.md`

## 3. Check Repository

After upload:

1. Open the repo in an incognito/private browser window.
2. Confirm the repo is visible without signing in.
3. Open `outputs/final_video.mp4` and confirm GitHub can display or download it.
4. Open `report/Final_Report_S11259035.pdf`.
5. Confirm `README.md` appears on the repo homepage.

## 4. Submit to Ecourse

Submit the full repository URL, for example:

```text
https://github.com/your-account/final-agentic-short-video-project
```

Do not submit:

- GitHub profile URL only
- single file URL
- Google Drive URL
- YouTube URL only
